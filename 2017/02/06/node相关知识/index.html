nrm

nrm 是一个管理 npm 源的工具。用过 ruby 和 gem 的同学会比较熟悉，通常我们会把 gem 源切到国内的淘宝镜像，这样在安装和更新一些包的时候比较快。nrm 同理，用来切换官方 npm 源和国内的 npm 源（如: cnpm），当然也可以用来切换官方 npm 源和公司私有 npm 源

require

  1. require 可加载 .js、.json 和 .node 后缀的文件

  2. require 的过程是同步的，所以这样是错误的:

require 这个文件得到的是空对象 {}

  1. require 目录的机制是:

    如果目录下有 package.json 并指定了 main 字段，则用之
如果不存在 package.json，则依次尝试加载目录下的 index.js 和 index.node

  2. require 过的文件会加载到缓存，所以多次 require 同一个文件（模块）不会重复加载

  3. 判断是否是程序的入口文件有两种方式:

require.main === module（推荐）

module.parent === null
