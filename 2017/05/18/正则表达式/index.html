正则规则

元字符

下面的表格是我们常用的一些的元字符。

代码 说明
. 匹配换行符以外的所有字符
\w 匹配字母或数字或下划线或汉字
\s 匹配任意的空白符
\d 匹配数字
\b 匹配单词的开始或结束
^ 匹配字符串的开始
$ 匹配字符串的结束

例子：

在hi him history 单词中匹配hi 只需要使用 \bhi\b
匹配0开头两位数字，然后-连接号，再后面8个数字的电话号码 0\d\d-\d\d\d\d\d\d\d\d 或者 0\d{2}-\d{8}
匹配5-8位QQ号码，^\d{5,8}$ {5,8}表示的意思重复次数不能少于5次，不能多于8次

字符转义

如果想查找元字符本身的话，比如要匹配.或* 就需要转义了，需要用\用来取消这些字符的特殊意义。应该使用\.和\*。当然如果要查找\本身，就使用\\

限定符

下面表格中是正则表达式中所有的限定符(指定数量代码)

代码/语法 说明
* 重复0次或者更多次
+ 重复1次或者更多次
？ 重复0次或者1次
{n} 重复n次
{n,} 重复n次 或者更多次
{n,m} 重复n 到 m 次

例子

windows/d+ 匹配windows后面跟一个或者多个数字
^\w+ 匹配一行的第一个单词

字符类

如果想匹配没有预定义元字符的字符组合，应该怎么做？
如匹配元音字母(a,e,i,o,u) 只需要用方括号列出来就可以了。。[aeiou]
[.?!]则是匹配标点符号(.或?或!)
[0-9]代表的意思是\d是一样的

分支条件

分支条件指的是有几种规则，如果满足其中一种任意规则，当应该当成匹配，具体方法是用|规则隔开
例子

0\d{2}-\d{8}|0\d{3}-\d{7} 这个表达式匹配两种连子号分隔的电话号码如(020-12345678或0123-1234567)

分组

前面已经知道怎么重复单个字符(使用限定符)，如果要重复多个字符。怎么实现？这里可以使用小括号来指定子表达式(也叫分组)，然后你也可以指定这个子表达式的重复次数
例子

(\d{3}\.){3}\d{3} 理解这个表达式。先看 \d{3}\. 指的是 匹配3个数字加上一个英文句号 , (\d{3}\.){3} 是 匹配3个数字加上一个英文句号 重复3遍。然后后面再跟随3个数字

反义

下面表格是常用的反义表格

代码/语法 说明
\W 匹配任意不是字母，数字，下划线，汉字的字符
\S 匹配任意不是空白符的字符
\D 匹配任意非数字的字符
\B 匹配不是单词开头或结束的位置
[^x] 匹配除x以外的任意字符
[^aeiou] 匹配除aeiou以外的任意字符

###后向引用
使用小括号指定一个子表达式后，匹配这个子表达式的文本(也就是此分组捕获的内容)可以在表达式或其它程序中作进一步的处理。默认情况下，每个分组会自动拥有一个组号，规则是：从左向右，以分组的左括号为标志，第一个出现的分组的组号为1，第二个为2，以此类推。
后向引用 用于重复搜索前面某个分组匹配的文本
\b(\w+)\b\s+\1\b 可以用来匹配重复的单词，像go go, 或者kitty kitty。

常用的分组语法

分类 代码 说明
捕获 (exp) 匹配exp,并捕获文本到自动命名的组里
捕获 (? < name > exp) 匹配exp,并捕获文本到名称为name的组里，也可以写成(?’name’exp)
捕获 (?:exp) 匹配exp,但不捕获，也不分配组号

零宽断言

分类 代码 说明
零宽断言 (?=exp) 匹配exp前面的位置
零宽断言 (?!exp) 匹配后面跟的不是exp的位置

贪婪与懒惰

当正则表达式中包含能接受重复的限定符时，通常的行为是（在使整个表达式能得到匹配的前提下）匹配尽可能多的字符。以这个表达式为例：a.*b，它将会匹配最长的以a开始，以b结束的字符串。如果用它来搜索aabab的话，它会匹配整个字符串aabab。这被称为贪婪匹配。

有时，我们更需要懒惰匹配，也就是匹配尽可能少的字符。前面给出的限定符都可以被转化为懒惰匹配模式，只要在它后面加上一个问号?。这样.*?就意味着匹配任意数量的重复，但是在能使整个匹配成功的前提下使用最少的重复。现在看看懒惰版的例子吧：

代码 说明
*？ 重复任意次，但尽可能少重复
+? 重复1次或更多次，但尽可能少重复
?？ 重复0次或1次，，但尽可能少重复
{n,m}？ 重复n-m次，，但尽可能少重复
{n,}？ 重复n-任意次，但尽可能少重复

js 中exec、test、match、search用法

RegExp.prototype.exec()

regexObj.exec(str);
如果匹配成功返回一个数组，匹配失败返回null

例子

对象 属性/index 描述 例子
result [0] 完全匹配的字符串 Quick Brown Fox Jumps
result [1] ,…., [n] 第1~n个匹配的子字符串 [1] = Brown [2] = Jumps
result index 匹配第一个字符的位置 4 (也就是q的位置)
result input 原始字符串 The Quick Brown Fox Jumps Over The Lazy Dog

RegExp.prototype.test()

regexObj.test(str)
如果匹配 返回true, 否则 false

例子

String.prototype.match();

str.match(regexp);
如果匹配成功，返回一个数组，失败则返回null
如果正则表达式没有g标签，结果返回与regObj.exec()的结果一样。 如果有g标签则返回一个数组包含包含满足所有子表达式而不是匹配对象的数组，捕获组不会返回
如果参数是string 或 number 将会隐形转换为RegExp。

例子

String.prototype.search()

str.search(regexp)
返回一个匹配的字符串的位置(index),如果没有发现返回 -1
如果参数是string 或 number 将会隐形转换为RegExp。
